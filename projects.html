Graph Algorithms in Java

View Graph 

When selecting a destination in Google Maps, the question probably arises "How does Google know the shortest route?". Essentially, it is a series of algorithms, akin to the term project I completed for MET CS 526 - Data Structures & Algorithms. For this project, I implemented two different algorithms to find the shortest path in a given graph

Suppose that the accompanying graph is a weighted, undirected graph representing a network of cities. Each node in the graph represents a city, with the edge between two nodes representing a road connecting the two cities, and the weight of an edge representing the distance between the two cities on the connecting road. Given an input node, both algorithms iteratively compute the next node in a shortest path

Essentially, one algorithm looks at all nodes that are adjacent to the given input node, determines the direct distance of each adjacent node to the destination node, and iteratively steps through each node until the destination is reached. The second algorithm is similar to the first, with the exception that the weight of two nodes is included in the equation

Database Creation - Conceptual & Physical 
Entity Relationship Diagrams (ERD's)

View Conceptual ERD  View Physical ERD 

Creating a database from scratch requires a great deal of planning and conceptualization. My term project for MET CS 699 - Database Design & Implementation for Business called for such a task, with the creation of conceptual, as well as physical entity relationship diagrams (ERD's)

When drafting these types of diagrams, the main components include the entities, their attributes, and the relationships between the entities. The relationships between the entities were represented using Crow's Foot notation, with the database structure including normalization where appropriate

As a car enthusiast personally, I opted to create a database for a fictional car dealership, with the typical entities/departments you would find including sales, service, and parts
